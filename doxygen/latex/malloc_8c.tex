{\footnotesize\ttfamily $\ast$ calloc (\begin{DoxyParamCaption}\item[{size\+\_\+t}]{count,  }\item[{size\+\_\+t}]{size }\end{DoxyParamCaption})}

This function is used to allocate and zero a memory region for use with data structures. Similar to the C standard library implementation of this function, this function accepts two arguments\+: a {\ttfamily count} or {\ttfamily nmemb} parameter which specifies the number of members of space to allocate, followed by a {\ttfamily size} parameter indicating the size of each member in bytes. Similarly to \mbox{\hyperlink{malloc_8c_a41a55f77c79e9f4122257e4177b6af0f}{malloc()}}, this function returns a {\ttfamily NULL} pointer if the allocation fails for any reason. \begin{DoxySeeAlso}{See also}

\doxysubsubsection{\texorpdfstring{malloc()}{malloc()}}
{\footnotesize\ttfamily $\ast$ malloc (\begin{DoxyParamCaption}\item[{size\+\_\+t}]{size }\end{DoxyParamCaption})}

This function wraps the EFI native {\ttfamily Boot\+Services-\/$>$Allocate\+Pool()} function to provide a more user-\/friendly memory allocation interface. This function allocates {\ttfamily size} bytes of memory under the UEFI {\ttfamily Efi\+Loader\+Data} class. Similar to the C standard library versions of {\ttfamily $\ast$malloc()}, this function returns {\ttfamily NULL} if, for any reason, the memory allocation request fails.
